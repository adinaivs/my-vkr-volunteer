# Phase 2: Руководство пользователя

## Управление участниками и назначение задач

### Для организаторов

#### 1. Просмотр участников проекта

После того как вы одобрили заявки волонтеров, они становятся участниками вашего проекта.

**Как посмотреть участников:**
1. Откройте страницу вашего проекта
2. Перейдите на вкладку **"Участники"**
3. Вы увидите список всех участников с информацией:
   - Имя и фамилия
   - Email и город
   - Дата присоединения
   - Количество назначенных задач

#### 2. Назначение задач участникам

**Способ 1: Через карточку участника**
1. На вкладке "Участники" найдите нужного волонтера
2. Нажмите кнопку **"Назначить задачу"**
3. В открывшемся окне выберите задачу из списка
4. Нажмите **"Назначить задачу"**

**Что происходит при назначении:**
- Волонтер получает задачу со статусом "Назначена"
- Счетчик волонтеров в задаче увеличивается
- Если набрано достаточно волонтеров, задача переходит в статус "В процессе"

#### 3. Просмотр задач участника

1. На вкладке "Участники" найдите нужного волонтера
2. Нажмите кнопку **"Показать задачи"**
3. Раскроется список всех назначенных задач с информацией:
   - Название и описание задачи
   - Статус (Назначена / Выполнена / Подтверждена)
   - Дата назначения

#### 4. Отмена назначения задачи

**Когда можно отменить:**
- Только задачи со статусом "Назначена"
- Нельзя отменить выполненные или подтвержденные задачи

**Как отменить:**
1. Раскройте список задач участника
2. Найдите нужную задачу
3. Нажмите кнопку **"Отменить"**
4. Подтвердите действие

**Что происходит при отмене:**
- Назначение получает статус "Отменено"
- Счетчик волонтеров в задаче уменьшается
- Запись о назначении сохраняется в истории

#### 5. Управление статусами проекта

Проект проходит через несколько этапов жизненного цикла:

**Статусы проекта:**
1. **Черновик** → Проект создан, но не отправлен на модерацию
2. **На модерации** → Проект ожидает одобрения админом
3. **Набор волонтеров** → Проект опубликован, идет прием заявок
4. **Скоро начнется** → Набор завершен, проект готов к старту
5. **Активный** → Проект выполняется
6. **Завершенный** → Проект успешно завершен
7. **Отмененный** → Проект отменен

**Допустимые переходы:**

**Из "Набор волонтеров":**
- → **Скоро начнется** (требуется минимум 1 участник)
- → **Отменен**

**Из "Скоро начнется":**
- → **Активный** (требуется наступление даты начала)
- → **Отменен**

**Из "Активный":**
- → **Завершенный** (требуется завершение всех задач)
- → **Отменен**

**Как изменить статус:**
1. Откройте страницу проекта
2. Используйте кнопку изменения статуса (будет добавлена в UI)
3. Выберите новый статус
4. Система проверит условия перехода

**Важно:**
- При отмене проекта все pending назначения автоматически отменяются
- Нельзя завершить проект, пока не завершены все задачи
- Нельзя активировать проект до даты начала

### Типичные сценарии использования

#### Сценарий 1: Запуск нового проекта
1. Создайте проект и добавьте задачи
2. Отправьте на модерацию
3. После одобрения проект переходит в статус "Набор волонтеров"
4. Одобряйте заявки волонтеров
5. Когда набрано достаточно участников, переведите в "Скоро начнется"
6. Назначьте задачи участникам
7. В дату начала переведите в "Активный"

#### Сценарий 2: Перераспределение задач
1. Откройте вкладку "Участники"
2. Найдите участника, которому нужно отменить задачу
3. Раскройте список его задач
4. Отмените ненужную задачу
5. Найдите другого участника
6. Назначьте ему эту задачу

#### Сценарий 3: Завершение проекта
1. Убедитесь, что все задачи выполнены или отменены
2. Переведите проект в статус "Завершенный"
3. Система зафиксирует завершение

### Ограничения и правила

**Назначение задач:**
- ✅ Можно назначить задачу только участнику проекта
- ✅ Одну задачу можно назначить одному волонтеру только один раз
- ✅ Задача должна принадлежать проекту
- ❌ Нельзя назначить задачу волонтеру, не являющемуся участником

**Отмена назначения:**
- ✅ Можно отменить задачу со статусом "Назначена"
- ❌ Нельзя отменить выполненную задачу
- ❌ Нельзя отменить подтвержденную задачу

**Изменение статуса:**
- ✅ Переходы должны следовать определенному порядку
- ❌ Нельзя пропускать этапы
- ❌ Нельзя вернуться к предыдущему статусу (кроме отмены)

### Уведомления

Система использует toast-уведомления для информирования о результатах операций:

- ✅ **Зеленые** - успешные операции
- ⚠️ **Желтые** - предупреждения
- ❌ **Красные** - ошибки

### Советы по эффективному использованию

1. **Планируйте заранее**: Создайте все задачи до начала набора волонтеров
2. **Назначайте постепенно**: Не спешите назначать все задачи сразу
3. **Следите за балансом**: Распределяйте задачи равномерно между участниками
4. **Используйте заметки**: Добавляйте заметки к участникам для лучшей организации
5. **Проверяйте статусы**: Регулярно проверяйте статусы задач и участников

### Часто задаваемые вопросы

**Q: Могу ли я назначить одну задачу нескольким волонтерам?**  
A: Да, если задача требует несколько волонтеров (указано в `requiredVolunteers`)

**Q: Что происходит с назначениями при отмене проекта?**  
A: Все pending назначения автоматически отменяются

**Q: Могу ли я изменить задачу после назначения?**  
A: Да, но это может повлиять на назначенных волонтеров

**Q: Как волонтер узнает о назначенной задаче?**  
A: В текущей версии волонтер видит назначенные задачи в своем личном кабинете

**Q: Можно ли вернуть отмененное назначение?**  
A: Нет, но можно создать новое назначение той же задачи
