# Уведомление о достижении квоты волонтеров

## Описание

Когда проект в статусе "Набор волонтеров" (recruiting) достигает нужного количества участников (`currentVolunteers >= maxVolunteers`), организатор получает уведомление с предложением завершить набор.

## Когда показывается уведомление

### Сценарий 1: При открытии страницы проекта

Если организатор открывает проект, который уже набрал нужное количество волонтеров:

**Условия:**
- Статус проекта: `recruiting`
- `currentVolunteers >= maxVolunteers`
- Первая загрузка страницы (не при обновлении)

**Уведомление:**
```
🎉 Набрано нужное количество волонтеров (X/Y)! 
Вы можете завершить набор и перейти к следующему этапу.
```

**Длительность:** 8 секунд

### Сценарий 2: После одобрения заявки

Если организатор одобряет заявку, и это приводит к достижению квоты:

**Условия:**
- Статус проекта: `recruiting`
- Одобрение заявки увеличивает `currentVolunteers` до `maxVolunteers` или больше

**Последовательность уведомлений:**
1. Сначала: "Заявка одобрена" (стандартное уведомление)
2. Через 1 секунду: "🎉 Набрано нужное количество волонтеров..."

**Длительность:** 8 секунд

## Технические детали

### Проверка при загрузке страницы

```typescript
// В функции fetchProject
if (!project && data.project.status === 'recruiting' && 
    data.project.currentVolunteers >= data.project.maxVolunteers) {
  toast.success(
    `🎉 Набрано нужное количество волонтеров (${data.project.currentVolunteers}/${data.project.maxVolunteers})! Вы можете завершить набор и перейти к следующему этапу.`,
    { duration: 8000 }
  );
}
```

**Условие `!project`** гарантирует, что уведомление показывается только при первой загрузке, а не при каждом обновлении данных.

### Проверка после одобрения заявки

```typescript
// В функции handleApplicationAction
if (action === 'approve' && project && project.status === 'recruiting') {
  const newCurrentVolunteers = project.currentVolunteers + 1;
  if (newCurrentVolunteers >= project.maxVolunteers) {
    setTimeout(() => {
      toast.success(
        `🎉 Набрано нужное количество волонтеров (${newCurrentVolunteers}/${project.maxVolunteers})! Вы можете завершить набор и перейти к следующему этапу.`,
        { duration: 8000 }
      );
    }, 1000);
  }
}
```

**`setTimeout(1000)`** создает задержку в 1 секунду между уведомлением об одобрении и уведомлением о достижении квоты.

## Пользовательский опыт

### Пример 1: Проект с квотой 10 волонтеров

1. Организатор создает проект с `maxVolunteers = 10`
2. Одобряет 9 заявок
3. Одобряет 10-ю заявку
4. Видит два уведомления:
   - ✅ "Заявка одобрена"
   - 🎉 "Набрано нужное количество волонтеров (10/10)! Вы можете завершить набор..."
5. Нажимает кнопку "Завершить набор"
6. Проект переходит в статус "Скоро начнется"

### Пример 2: Возвращение к проекту

1. Организатор одобрил 10 заявок вчера
2. Сегодня открывает страницу проекта
3. Видит уведомление:
   - 🎉 "Набрано нужное количество волонтеров (10/10)! Вы можете завершить набор..."
4. Может завершить набор или продолжить одобрять заявки (если хочет больше волонтеров)

### Пример 3: Превышение квоты

1. Проект с `maxVolunteers = 10`
2. Организатор одобрил 12 заявок
3. При открытии страницы видит:
   - 🎉 "Набрано нужное количество волонтеров (12/10)! Вы можете завершить набор..."
4. Может завершить набор с 12 волонтерами

## Визуальное оформление

**Тип уведомления:** Success (зеленое)

**Иконка:** 🎉 (эмодзи праздника)

**Цвет:** Зеленый фон с белым текстом

**Позиция:** Верхний правый угол (стандартная позиция toast)

**Длительность:** 8 секунд (дольше обычного, чтобы организатор успел прочитать)

## Поведение

### Уведомление НЕ показывается если:
- ❌ Статус проекта не `recruiting`
- ❌ `currentVolunteers < maxVolunteers`
- ❌ Страница обновляется (не первая загрузка)
- ❌ Заявка отклонена (не одобрена)

### Уведомление показывается если:
- ✅ Статус проекта `recruiting`
- ✅ `currentVolunteers >= maxVolunteers`
- ✅ Первая загрузка страницы ИЛИ одобрение заявки

## Дальнейшие действия организатора

После получения уведомления организатор может:

1. **Завершить набор немедленно:**
   - Нажать кнопку "Завершить набор" (синяя)
   - Проект перейдет в статус "Скоро начнется"

2. **Продолжить набор:**
   - Одобрить еще заявки (если хочет больше волонтеров)
   - Система позволяет превышать `maxVolunteers`

3. **Назначить задачи:**
   - Перейти на вкладку "Участники"
   - Назначить задачи волонтерам
   - Затем завершить набор

## Преимущества

✅ **Информирование:** Организатор сразу знает, что квота достигнута

✅ **Гибкость:** Можно завершить набор или продолжить одобрять заявки

✅ **Своевременность:** Уведомление показывается в момент достижения квоты

✅ **Ненавязчивость:** Уведомление исчезает через 8 секунд, не блокирует работу

✅ **Контекст:** Показывает точные цифры (X/Y волонтеров)

## Ограничения

⚠️ **Нет email уведомлений:** Уведомление показывается только в UI

⚠️ **Нет истории:** Уведомление не сохраняется, показывается один раз

⚠️ **Только для recruiting:** Не работает для других статусов

⚠️ **Нет напоминаний:** Если организатор закрыл уведомление, оно не повторится

## Возможные улучшения

### Высокий приоритет:
1. **Email уведомление:**
   - Отправлять email когда квота достигнута
   - Полезно, если организатор не в системе

2. **Уведомления в личном кабинете:**
   - Сохранять уведомление в истории
   - Показывать в разделе "Уведомления"

### Средний приоритет:
3. **Прогресс-бар:**
   - Показывать прогресс набора (80%, 90%, 100%)
   - Визуальный индикатор приближения к квоте

4. **Настройки уведомлений:**
   - Позволить организатору настроить порог (например, уведомлять при 80%)
   - Отключить уведомления, если не нужны

### Низкий приоритет:
5. **Push уведомления:**
   - Browser push notifications
   - Работают даже когда вкладка закрыта

6. **Telegram/SMS уведомления:**
   - Интеграция с мессенджерами
   - Мгновенное оповещение
