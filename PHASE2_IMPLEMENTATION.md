# Phase 2: Управление участниками и назначение задач - Реализация

## Статус: ✅ Завершено

## Реализованные API Endpoints

### 1. GET /api/organizer/projects/[id]/participants
**Описание:** Получение списка участников проекта

**Доступ:** Только организатор проекта или админ

**Ответ:**
```json
[
  {
    "id": "uuid",
    "volunteerId": "uuid",
    "firstName": "string",
    "lastName": "string",
    "email": "string",
    "phone": "string",
    "avatarUrl": "string",
    "city": "string",
    "joinedAt": "timestamp",
    "isActive": boolean,
    "notes": "string",
    "assignedTasksCount": number
  }
]
```

### 2. POST /api/organizer/projects/[projectId]/tasks/[taskId]/assign
**Описание:** Назначение задачи участнику проекта

**Доступ:** Только организатор проекта или админ

**Тело запроса:**
```json
{
  "volunteerId": "uuid"
}
```

**Валидация:**
- Волонтер должен быть активным участником проекта
- Задача должна принадлежать проекту
- Задача не должна быть уже назначена этому волонтеру

**Побочные эффекты:**
- Увеличивает `task.currentVolunteers` на 1
- Если `currentVolunteers >= requiredVolunteers`, меняет `task.status` на `in_progress`

### 3. DELETE /api/organizer/projects/[projectId]/tasks/[taskId]/assignments/[volunteerId]
**Описание:** Отмена назначения задачи

**Доступ:** Только организатор проекта или админ

**Валидация:**
- Нельзя отменить задачи со статусом `completed` или `confirmed`

**Побочные эффекты:**
- Меняет статус назначения на `cancelled` (soft delete)
- Уменьшает `task.currentVolunteers` на 1

### 4. PATCH /api/organizer/projects/[id]/status
**Описание:** Изменение статуса проекта

**Доступ:** Только организатор проекта или админ

**Тело запроса:**
```json
{
  "newStatus": "recruiting" | "upcoming" | "active" | "completed" | "cancelled"
}
```

**Допустимые переходы:**
- `recruiting` → `upcoming` (требуется минимум 1 участник)
- `upcoming` → `active` (требуется `startDate <= текущая дата`)
- `active` → `completed` (требуется все задачи завершены или отменены)
- `active` → `cancelled`
- `recruiting` → `cancelled`
- `upcoming` → `cancelled`

**Побочные эффекты:**
- При переходе в `cancelled` все pending назначения меняются на `cancelled`

## Реализованные UI компоненты

### 1. Вкладка "Участники" (Participants Tab)
**Расположение:** `app/organizer/projects/[id]/page.tsx`

**Функциональность:**
- Отображение списка всех участников проекта
- Показ количества назначенных задач для каждого участника
- Кнопка "Показать задачи" для просмотра назначенных задач
- Кнопка "Назначить задачу" для открытия модального окна

### 2. Модальное окно назначения задачи (Task Assignment Modal)
**Функциональность:**
- Выбор задачи из списка доступных задач проекта
- Визуальное выделение выбранной задачи
- Отображение информации о задаче (название, описание, требуемые навыки, статус)
- Кнопка "Назначить задачу" для подтверждения

### 3. Раскрывающийся список задач участника
**Функциональность:**
- Отображение всех назначенных задач участника
- Показ статуса каждой задачи (назначена, выполнена, подтверждена)
- Дата назначения задачи
- Кнопка "Отменить" для отмены назначения (только для статуса "assigned")

## Технические детали

### Транзакции
Все операции изменения данных используют Prisma транзакции для обеспечения консистентности:
- Назначение задачи: создание TaskAssignment + обновление currentVolunteers
- Отмена назначения: обновление статуса + уменьшение currentVolunteers
- Изменение статуса проекта: обновление проекта + каскадное обновление назначений

### Валидация прав доступа
Все endpoints проверяют:
1. Наличие валидной сессии (401 если нет)
2. Роль пользователя (403 для волонтеров)
3. Ownership проекта (403 если не организатор и не админ)

### Блокировка скролла
При открытии модальных окон (карта или назначение задачи) скролл страницы блокируется через `document.body.style.overflow = 'hidden'`

### Toast-уведомления
Все операции используют toast-уведомления вместо alert():
- Успешное назначение задачи
- Ошибки валидации
- Подтверждение отмены назначения

## Тестирование

### Ручное тестирование
1. Создайте проект и опубликуйте его
2. Одобрите несколько заявок волонтеров
3. Перейдите на вкладку "Участники"
4. Назначьте задачи участникам
5. Проверьте отображение назначенных задач
6. Отмените назначение задачи
7. Измените статус проекта

### Проверка валидации
- Попытка назначить задачу дважды одному волонтеру
- Попытка отменить завершенную задачу
- Попытка изменить статус без выполнения условий
- Попытка доступа к чужому проекту
