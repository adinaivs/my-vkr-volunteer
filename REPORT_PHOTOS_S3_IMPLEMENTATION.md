# Реализация загрузки фотографий отчетов в S3

## Что было изменено

### Проблема
Ранее фотографии отчетов о выполнении задач конвертировались в base64 и сохранялись напрямую в базу данных PostgreSQL. Это приводило к:
- Увеличению размера базы данных
- Медленной загрузке данных
- Неэффективному использованию ресурсов

### Решение
Фотографии теперь загружаются в S3 хранилище (Timeweb Cloud Storage), как и другие файлы (документы регистрации, фотографии проектов).

## Изменения в коде

### 1. Новый API endpoint для загрузки фотографий
**Файл:** `app/api/upload/report-photos/route.ts`

- **Метод:** POST
- **URL:** `/api/upload/report-photos`
- **Доступ:** Только для волонтеров
- **Параметры:** FormData с полем `photos` (массив файлов)
- **Ограничения:**
  - Максимум 10 фотографий за раз
  - Максимальный размер каждого файла: 5 МБ
  - Разрешенные форматы: JPG, PNG, WEBP
- **Возвращает:** Массив URL загруженных файлов в S3

### 2. Обновление фронтенда
**Файл:** `app/volunteer/my-projects/page.tsx`

Функция `handlePhotoUpload` теперь:
1. Проверяет файлы на клиенте (размер, тип, количество)
2. Создает FormData с файлами
3. Отправляет POST запрос на `/api/upload/report-photos`
4. Получает URL файлов из S3
5. Сохраняет URL в состоянии компонента

### 3. Обновление библиотеки S3
**Файл:** `lib/s3.ts`

Добавлена поддержка формата WEBP в функции `validateFile`.

## Структура хранения в S3

```
voluteerkr/
├── verification-docs/     # Документы о регистрации организаций
├── project-images/        # Фотографии проектов
└── report-photos/         # Фотографии отчетов (НОВОЕ)
    ├── 1234567890-abc123.jpg
    ├── 1234567891-def456.png
    └── ...
```

## Формат URL фотографий

```
https://s3.twcstorage.ru/voluteerkr/report-photos/1234567890-abc123.jpg
```

## Настройки S3 (из .env)

```env
S3_URL="https://s3.twcstorage.ru"
BUCKET_NAME="voluteerkr"
S3_ACCESS_KEY="..."
S3_SECRET_ACCESS_KEY="..."
REGION="ru-1"
```

## Преимущества нового подхода

1. ✅ **Уменьшение размера БД** - фотографии не хранятся в PostgreSQL
2. ✅ **Быстрая загрузка** - S3 оптимизирован для хранения файлов
3. ✅ **Масштабируемость** - легко увеличить объем хранилища
4. ✅ **CDN поддержка** - возможность подключить CDN для быстрой доставки
5. ✅ **Единообразие** - все файлы (документы, фото проектов, фото отчетов) хранятся в одном месте

## Обратная совместимость

API endpoint для создания отчета (`/api/volunteer/tasks/[assignmentId]/report`) не изменился:
- Принимает массив URL фотографий в поле `photos`
- Сохраняет URL в базе данных (не сами файлы)

Старые отчеты с base64 фотографиями продолжат работать, но новые отчеты будут использовать S3 URL.

## Тестирование

1. Войдите как волонтер
2. Перейдите в "Мои проекты"
3. Выберите задачу со статусом "assigned"
4. Нажмите "Отправить отчёт"
5. Загрузите фотографии (до 10 штук, до 5 МБ каждая)
6. Проверьте, что фотографии отображаются в превью
7. Отправьте отчет
8. Проверьте в S3 бакете наличие файлов в папке `report-photos/`

## Безопасность

- ✅ Только авторизованные волонтеры могут загружать фотографии
- ✅ Валидация типа и размера файлов на сервере
- ✅ Уникальные имена файлов (timestamp + random string)
- ✅ Ограничение количества файлов за раз (10 штук)
