# Phase 3 & 4: Управление статусами проекта - Реализация

## Статус: ✅ Завершено

## Что реализовано

### UI для управления статусами проекта

Добавлен интерфейс для изменения статусов проекта на странице деталей проекта организатора.

## Функциональность

### 1. Кнопки управления статусами

**Расположение:** Левая колонка страницы проекта, под бейджем статуса

**Отображаются для статусов:**
- **recruiting** (Набор волонтеров)
- **upcoming** (Скоро начнется)
- **active** (Активный)

**Не отображаются для:**
- draft (Черновик)
- moderation (На модерации)
- rejected (Отклонен)
- completed (Завершен)
- cancelled (Отменен)
- blocked (Заблокирован)

### 2. Доступные переходы статусов

#### Из статуса "Набор волонтеров" (recruiting):
- ✅ **Завершить набор** → переход в "Скоро начнется" (upcoming)
  - Цвет: Синий
  - Требование: минимум 1 активный участник
- ❌ **Отменить проект** → переход в "Отменен" (cancelled)
  - Цвет: Красный
  - Эффект: все pending назначения отменяются

#### Из статуса "Скоро начнется" (upcoming):
- ✅ **Активировать проект** → переход в "Активный" (active)
  - Цвет: Фиолетовый
  - Требование: дата начала проекта <= текущая дата
- ❌ **Отменить проект** → переход в "Отменен" (cancelled)
  - Цвет: Красный

#### Из статуса "Активный" (active):
- ✅ **Завершить проект** → переход в "Завершенный" (completed)
  - Цвет: Зеленый
  - Требование: все задачи завершены или отменены
- ❌ **Отменить проект** → переход в "Отменен" (cancelled)
  - Цвет: Красный

### 3. Подтверждение действий

Перед изменением статуса пользователю показывается модальное окно подтверждения с описанием действия:

**Для "Завершить набор":**
> "Вы уверены, что хотите завершить набор волонтеров и перевести проект в статус "Скоро начнется"?"

**Для "Активировать проект":**
> "Вы уверены, что хотите активировать проект? После этого он будет в процессе выполнения."

**Для "Завершить проект":**
> "Вы уверены, что хотите завершить проект? Убедитесь, что все задачи выполнены."

**Для "Отменить проект":**
> "Вы уверены, что хотите отменить проект? Все назначения задач будут отменены."

### 4. Индикация процесса

Во время изменения статуса:
- Кнопки становятся неактивными
- Показывается спиннер загрузки
- Текст меняется на "Изменение..."

### 5. Уведомления

После успешного изменения статуса:
- ✅ Зеленое toast-уведомление: "Статус проекта изменен на [новый статус]"
- Автоматическое обновление данных проекта

При ошибке:
- ❌ Красное toast-уведомление с описанием ошибки

## Технические детали

### Новые функции

#### `handleChangeStatus(newStatus: string)`
Обрабатывает изменение статуса проекта:
1. Показывает модальное окно подтверждения
2. Отправляет PATCH запрос на `/api/organizer/projects/[id]/status`
3. Обновляет данные проекта после успешного изменения
4. Показывает toast-уведомление

#### `getAvailableStatusTransitions(currentStatus: string)`
Возвращает массив доступных переходов для текущего статуса:
```typescript
Array<{
  status: string;      // Новый статус
  label: string;       // Текст кнопки
  color: string;       // CSS классы для цвета кнопки
}>
```

### Состояния

```typescript
const [changingStatus, setChangingStatus] = useState(false);
```

Используется для:
- Блокировки кнопок во время запроса
- Отображения индикатора загрузки

### API Endpoint

**Используется:** `PATCH /api/organizer/projects/[id]/status`

**Тело запроса:**
```json
{
  "newStatus": "upcoming" | "active" | "completed" | "cancelled"
}
```

**Ответ при успехе:**
```json
{
  "message": "Статус проекта обновлен",
  "project": { ... }
}
```

**Ответ при ошибке:**
```json
{
  "error": "Описание ошибки",
  "currentStatus": "recruiting",
  "allowedTransitions": ["upcoming", "cancelled"]
}
```

## Валидация на бэкенде

API endpoint выполняет следующие проверки:

1. **Аутентификация:** Пользователь должен быть авторизован
2. **Авторизация:** Только организатор проекта или админ
3. **Допустимость перехода:** Проверка по графу переходов
4. **Бизнес-правила:**
   - recruiting → upcoming: минимум 1 активный участник
   - upcoming → active: startDate <= текущая дата
   - active → completed: все задачи завершены или отменены

## Пользовательский сценарий

### Сценарий 1: Завершение набора волонтеров

1. Организатор открывает проект со статусом "Набор волонтеров"
2. Видит кнопку "Завершить набор" (синяя)
3. Нажимает на кнопку
4. Видит модальное окно подтверждения
5. Подтверждает действие
6. Видит индикатор загрузки
7. Получает уведомление об успехе
8. Статус проекта меняется на "Скоро начнется"
9. Кнопки обновляются: теперь доступны "Активировать проект" и "Отменить проект"

### Сценарий 2: Активация проекта

1. Организатор открывает проект со статусом "Скоро начнется"
2. Видит кнопку "Активировать проект" (фиолетовая)
3. Нажимает на кнопку
4. Если дата начала еще не наступила:
   - Получает ошибку: "Нельзя перейти в статус 'Активный' до даты начала проекта"
5. Если дата начала наступила:
   - Видит модальное окно подтверждения
   - Подтверждает действие
   - Статус меняется на "Активный"

### Сценарий 3: Завершение проекта

1. Организатор открывает проект со статусом "Активный"
2. Видит кнопку "Завершить проект" (зеленая)
3. Нажимает на кнопку
4. Если есть незавершенные задачи:
   - Получает ошибку: "Нельзя завершить проект, пока не завершены все задачи"
5. Если все задачи завершены:
   - Видит модальное окно подтверждения
   - Подтверждает действие
   - Статус меняется на "Завершенный"
   - Кнопки управления статусами исчезают

### Сценарий 4: Отмена проекта

1. Организатор открывает проект (любой активный статус)
2. Видит кнопку "Отменить проект" (красная)
3. Нажимает на кнопку
4. Видит предупреждение: "Все назначения задач будут отменены"
5. Подтверждает действие
6. Статус меняется на "Отменен"
7. Все pending назначения задач автоматически отменяются
8. Кнопки управления статусами исчезают

## Известные ограничения

1. **Automatic transitions не реализованы:**
   - Проект не переходит автоматически из upcoming в active при наступлении даты
   - Организатор должен вручную активировать проект

2. **Нет истории изменений статусов:**
   - Не сохраняется, кто и когда менял статус
   - Только timestamp в `updatedAt`

3. **Нет уведомлений волонтерам:**
   - Волонтеры не получают уведомления об изменении статуса проекта
