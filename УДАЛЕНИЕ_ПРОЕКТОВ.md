# Функционал удаления проектов

## Описание
Реализован функционал удаления проектов с разными правами доступа для организаторов и администраторов.

## Права доступа

### Организаторы
Организаторы могут удалять **только свои проекты** со следующими статусами:
- ✅ **Черновик** (`draft`) - проект еще не отправлен на модерацию
- ✅ **На модерации** (`moderation`) - проект находится на рассмотрении у администратора

**Ограничения:**
- ❌ Нельзя удалить опубликованные проекты (`published`)
- ❌ Нельзя удалить завершенные проекты (`completed`)
- ❌ Нельзя удалить отклоненные проекты (`rejected`)
- ❌ Нельзя удалить заблокированные проекты (`blocked`)
- ❌ Нельзя удалить чужие проекты

### Администраторы
Администраторы могут удалять **любые проекты** с **любым статусом**:
- ✅ Черновики (`draft`)
- ✅ На модерации (`moderation`)
- ✅ Опубликованные (`published`)
- ✅ Завершенные (`completed`)
- ✅ Отклоненные (`rejected`)
- ✅ Заблокированные (`blocked`)

## Где находится функционал

### Для организаторов
1. Перейдите в раздел **"Мои проекты"**
2. Нажмите на проект, чтобы открыть детали
3. Для черновиков и проектов на модерации появится красная кнопка **"Удалить проект"**

### Для администраторов
1. Перейдите в раздел **"Модерация проектов"**
2. Нажмите на любой проект, чтобы открыть детали
3. Внизу модального окна появится красная кнопка **"Удалить проект"**

## API эндпоинты

### Для организаторов
```
DELETE /api/projects/[id]
```
- Требуется авторизация
- Проверяет владельца проекта
- Разрешает удаление только для статусов `draft` и `moderation`

### Для администраторов
```
DELETE /api/admin/projects/[id]
```
- Требуется авторизация с ролью `admin`
- Разрешает удаление проектов с любым статусом
- Логирует действия администратора

## Безопасность

### Каскадное удаление
При удалении проекта автоматически удаляются:
- Все задачи проекта
- Все заявки волонтеров на проект
- Все связанные данные (настроено в Prisma схеме через `onDelete: Cascade`)

### Подтверждение действия
Перед удалением проекта пользователю показывается диалоговое окно с подтверждением:
> "Вы уверены, что хотите удалить этот проект? Это действие нельзя отменить."

### Проверки безопасности
- ✅ Проверка авторизации пользователя
- ✅ Проверка роли (для админского API)
- ✅ Проверка владельца проекта (для организаторов)
- ✅ Проверка допустимого статуса (для организаторов)
- ✅ Логирование действий администратора

## Технические детали

### Файлы, которые были изменены/созданы:

1. **Backend API:**
   - `app/api/projects/[id]/route.ts` - добавлен метод DELETE для организаторов
   - `app/api/admin/projects/[id]/route.ts` - создан новый файл с методом DELETE для админов

2. **Frontend:**
   - `app/organizer/projects/page.tsx` - добавлена функция `handleDeleteProject` и кнопки удаления
   - `app/admin/projects/page.tsx` - добавлена кнопка удаления в модальное окно

### Пример использования API

#### Организатор удаляет свой черновик:
```javascript
const response = await fetch(`/api/projects/${projectId}`, {
  method: 'DELETE',
});

if (response.ok) {
  console.log('Проект успешно удален');
}
```

#### Админ удаляет любой проект:
```javascript
const response = await fetch(`/api/admin/projects/${projectId}`, {
  method: 'DELETE',
});

if (response.ok) {
  console.log('Проект успешно удален администратором');
}
```

## Сообщения об ошибках

### Для организаторов:
- `401` - "Не авторизован"
- `403` - "Доступ запрещен" (не владелец проекта)
- `404` - "Проект не найден"
- `400` - "Можно удалять только черновики и проекты на модерации"
- `500` - "Ошибка при удалении проекта"

### Для администраторов:
- `401` - "Не авторизован"
- `403` - "Доступ запрещен" (не администратор)
- `404` - "Проект не найден"
- `500` - "Ошибка при удалении проекта"
