### Технические детали

#### Стек технологий
- Next.js 16.2.4
- TypeScript
- Prisma ORM
- PostgreSQL
- JWT (jose)
- bcryptjs
- Telegram Bot API

#### Архитектура
- Server Components для оптимизации производительности
- Client Components для интерактивных элементов
- API Routes для backend логики
- Middleware для защиты маршрутов
- Централизованная система переводов

#### Безопасность
- Хеширование паролей с bcrypt (10 раундов)
- JWT токены с истечением через 7 дней
- HttpOnly cookies для хранения токенов
- Временное хранилище кодов 2FA (рекомендуется Redis в продакшене)
- Валидация ролей на уровне middleware и API

### Известные ограничения

1. **Хранилище кодов 2FA**: Используется Map в памяти. В продакшене рекомендуется Redis.
2. **Rate limiting**: Отсутствует защита от брутфорса. Рекомендуется добавить в продакшене.
3. **Логирование**: Базовое логирование через console.log. Рекомендуется использовать Winston или Pino.

### Миграция с предыдущих версий

Если у вас уже есть пользователи в системе:

1. Примените миграцию: `npx prisma migrate deploy`
2. Обновите роль существующего пользователя на `admin`
3. Настройте Telegram через `npm run setup-telegram`
