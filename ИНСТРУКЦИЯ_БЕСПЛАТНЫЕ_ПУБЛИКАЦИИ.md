# Инструкция: Бесплатные публикации для организаторов

## Что реализовано

✅ **Админ может регулировать количество бесплатных публикаций** - на главной странице админ-панели есть блок для настройки количества бесплатных публикаций, которое получает каждый новый организатор

✅ **Организатор видит остаток бесплатных публикаций** - на дашборде организатора отображается карточка с количеством оставшихся бесплатных публикаций

✅ **Правильный подсчет** - бесплатные публикации уменьшаются только когда проект **публикуется** (одобряется администратором), а не когда отправляется на модерацию или отклоняется

## Быстрый старт

### 1. Применить настройку в базе данных

```bash
npx tsx scripts/apply-free-posts-setting.ts
```

### 2. Перезапустить приложение

```bash
npm run dev
```

## Как использовать

### Для администратора

1. Откройте `/admin/dashboard`
2. В верхней части страницы найдите зеленый блок "Бесплатные публикации для организаторов"
3. Нажмите "Изменить"
4. Введите нужное количество (например, 5)
5. Нажмите "Сохранить"

Теперь все **новые** организаторы при одобрении будут получать это количество бесплатных публикаций.

### Для организатора

1. После одобрения администратором вы получаете бесплатные публикации
2. На дашборде (`/organizer/dashboard`) видна зеленая карточка с количеством оставшихся публикаций
3. Создавайте проекты и отправляйте на модерацию - счетчик пока не меняется
4. Когда администратор одобрит проект и он опубликуется - счетчик уменьшится на 1
5. Если проект отклонят - счетчик не изменится, можете исправить и отправить снова

## Логика работы

| Действие | Счетчик бесплатных публикаций |
|----------|-------------------------------|
| Организатор одобрен админом | Устанавливается значение из настроек (по умолчанию 3) |
| Проект отправлен на модерацию | Не изменяется ❌ |
| Проект одобрен и опубликован | Уменьшается на 1 ✅ |
| Проект отклонен | Не изменяется ❌ |

## Примеры

### Пример 1: Успешная публикация
1. У организатора 3 бесплатные публикации
2. Создает проект → отправляет на модерацию → остается 3
3. Админ одобряет → проект публикуется → остается 2
4. Создает второй проект → отправляет → остается 2
5. Админ одобряет → публикуется → остается 1

### Пример 2: Отклонение проекта
1. У организатора 3 бесплатные публикации
2. Создает проект → отправляет на модерацию → остается 3
3. Админ отклоняет → остается 3 (не уменьшилось!)
4. Исправляет проект → отправляет снова → остается 3
5. Админ одобряет → публикуется → остается 2

## Что дальше?

После применения изменений:
- Зайдите в админ-панель и установите нужное количество бесплатных публикаций
- Одобрите нового организатора и проверьте, что у него появились бесплатные публикации
- Проверьте, что счетчик уменьшается только при публикации проекта
