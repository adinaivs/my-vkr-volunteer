# Changelog - Исправление логики регистрации

## Проблема
После регистрации волонтера показывалось сообщение о необходимости проверки администратором, хотя это должно быть только для организаторов.

## Решение

### 1. Обновлена схема базы данных
**Файл:** `prisma/schema.prisma`

Добавлены опциональные поля для организатора в таблицу `EmailVerificationToken`:
- `organizationName` - название организации
- `inn` - ИНН (14 цифр)
- `okpo` - ОКПО (8 цифр)
- `legalAddress` - юридический адрес
- `actualAddress` - фактический адрес
- `verificationDocUrl` - URL документа верификации

### 2. Обновлен API регистрации организатора
**Файл:** `app/api/auth/register/organizer/route.ts`

Теперь сохраняет все данные организатора в токене верификации, чтобы после подтверждения email создать полный профиль организатора.

### 3. Обновлен API верификации email
**Файл:** `app/api/auth/verify-email/route.ts`

Добавлена логика создания профиля организатора с данными из токена верификации:
```typescript
if (user.role === 'organizer') {
  await prisma.organizerProfile.create({
    data: {
      userId: user.id,
      organizationName: verificationToken.organizationName!,
      inn: verificationToken.inn!,
      okpo: verificationToken.okpo!,
      legalAddress: verificationToken.legalAddress || 'Не указан',
      actualAddress: verificationToken.actualAddress || 'Не указан',
      verificationStatus: 'pending',
      verificationDocUrl: verificationToken.verificationDocUrl,
    },
  });
}
```

API теперь возвращает флаг `needsVerification`:
```typescript
{
  user: { ... },
  needsVerification: user.role === 'organizer'
}
```

### 4. Обновлена страница верификации
**Файл:** `app/register/verify/page.tsx`

Добавлена логика перенаправления в зависимости от роли:
- **Волонтер** → `/dashboard` (сразу в личный кабинет)
- **Организатор** → `/register/success?type=organizer` (страница с информацией о проверке)

### 5. Обновлена страница успешной регистрации
**Файл:** `app/register/success/page.tsx`

Теперь показывает разные сообщения в зависимости от параметра `type`:

#### Для волонтера:
- ✅ Зеленое сообщение "Добро пожаловать!"
- Кнопка "Перейти в личный кабинет" (зеленая)
- Список действий: заполнить профиль, просмотреть проекты, подать заявки

#### Для организатора:
- ℹ️ Синее сообщение о проверке администратором
- Информация о времени проверки (1-3 дня)
- Кнопка "Перейти в личный кабинет" (синяя)
- Список действий: заполнить информацию, дождаться подтверждения

## Миграция базы данных

Применена миграция:
```bash
npx prisma migrate dev --name add_organizer_fields_to_verification
```

## Результат

✅ **Волонтеры** после подтверждения email сразу попадают в дашборд  
✅ **Организаторы** после подтверждения email видят сообщение о необходимости проверки администратором  
✅ Все данные организатора сохраняются и используются при создании профиля  
✅ Логика разделена и работает корректно для обеих ролей

## Тестирование

### Регистрация волонтера:

2. Введите код из email на `/register/verify`
3. ✅ Вы будете перенаправлены в `/dashboard`

### Регистрация организатора:
1. Заполните форму на `/register/organizer`
2. Введите код из email на `/register/verify`
3. ✅ Вы увидите страницу с информацией о проверке администратором
4. ✅ Профиль организатора создан со статусом "pending"
