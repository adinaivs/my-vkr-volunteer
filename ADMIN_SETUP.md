# Настройка админ-панели

## Обзор

Админ-панель защищена двухфакторной аутентификацией через Telegram. Администратор входит с помощью email и пароля из таблицы `users`, после чего получает 6-значный код в Telegram для подтверждения входа.

## Требования

1. Пользователь с ролью `admin` в таблице `users`
2. Telegram бот
3. Telegram User ID администратора

## Шаг 1: Создание Telegram бота

1. Откройте Telegram и найдите [@BotFather](https://t.me/BotFather)
2. Отправьте команду `/newbot`
3. Следуйте инструкциям для создания бота
4. Сохраните токен бота (например: `1234567890:ABCdefGHIjklMNOpqrsTUVwxyz`)

## Шаг 2: Получение Telegram User ID

1. Откройте Telegram и найдите [@userinfobot](https://t.me/userinfobot)
2. Отправьте команду `/start`
3. Бот отправит вам ваш User ID (например: `123456789`)

## Шаг 3: Настройка базы данных

### 3.1 Применить миграцию

```bash
npx prisma migrate deploy
```

### 3.2 Обновить настройки Telegram

Выполните SQL запрос для обновления настроек:

```sql
-- Обновить токен бота
UPDATE "settings" 
SET "value" = 'ВАШ_ТОКЕН_БОТА' 
WHERE "key" = 'ADMIN_TELEGRAM_BOT_TOKEN';

-- Обновить User ID
UPDATE "settings" 
SET "value" = 'ВАШ_TELEGRAM_USER_ID' 
WHERE "key" = 'ADMIN_TELEGRAM_USER_ID';
```

Или через Prisma Studio:

```bash
npx prisma studio
```

Откройте таблицу `Setting` и обновите значения:
- `ADMIN_TELEGRAM_BOT_TOKEN` → Ваш токен бота
- `ADMIN_TELEGRAM_USER_ID` → Ваш Telegram User ID

### 3.3 Создать пользователя-администратора

Если у вас еще нет пользователя с ролью `admin`, создайте его:

```sql
-- Создать администратора
INSERT INTO "users" (
  "id",
  "email",
  "phone",
  "password_hash",
  "role",
  "first_name",
  "last_name",
  "city",
  "status"
) VALUES (
  gen_random_uuid(),
  'admin@volonterkr.kg',
  '+996555000000',
  '$2a$10$...',  -- Хеш пароля (используйте bcrypt)
  'admin',
  'Администратор',
  'Системы',
  'Бишкек',
  'active'
);
```

Для генерации хеша пароля используйте Node.js:

```javascript
const bcrypt = require('bcryptjs');
const password = 'ваш_пароль';
const hash = bcrypt.hashSync(password, 10);
console.log(hash);
```

## Шаг 4: Тестирование

1. Откройте браузер и перейдите на `/admin/login`
2. Введите email и пароль администратора
3. Нажмите "Войти"
4. Проверьте Telegram - вы должны получить 6-значный код
5. Введите код на странице подтверждения
6. Вы будете перенаправлены на `/admin/dashboard`

## Безопасность

### Защита маршрутов

Middleware автоматически защищает следующие маршруты:

- `/admin/dashboard` - Главная страница админа
- `/admin/users` - Управление пользователями
- `/admin/projects` - Управление проектами
- `/admin/verifications` - Проверка организаторов
- `/admin/reports` - Отчеты
- `/admin/settings` - Настройки системы

### Разделение доступа

- **Администраторы** не могут заходить на страницы обычных пользователей (`/volunteer`, `/organizer`, `/dashboard`)
- **Обычные пользователи** не могут заходить на страницы админа (`/admin/*`)
- **Неавторизованные пользователи** перенаправляются на `/admin/login` при попытке доступа к админ-панели

### Срок действия кода

- Код 2FA действителен **5 минут**
- После использования код удаляется
- Можно запросить новый код через кнопку "Отправить код повторно"

- Русский (`ru`)
- Кыргызский (`kg`)

- `app/i18n/locales/ru/admin.json`
- `app/i18n/locales/kg/admin.json`

## Troubleshooting

### Код не приходит в Telegram

1. Проверьте правильность токена бота в таблице `settings`
2. Убедитесь, что вы начали диалог с ботом (отправьте `/start`)
3. Проверьте правильность User ID в таблице `settings`
4. Проверьте логи сервера на наличие ошибок

### Ошибка "У вас нет прав администратора"

1. Проверьте роль пользователя в таблице `users`
2. Убедитесь, что поле `role` имеет значение `admin`

### Ошибка "Код истек"

1. Код действителен только 5 минут
2. Запросите новый код через кнопку "Отправить код повторно"

### Не могу войти на обычные страницы после входа как админ

Это нормальное поведение. Администраторы имеют доступ только к админ-панели и не могут заходить на страницы обычных пользователей. Это сделано для безопасности и разделения ролей.

## Производственная среда

В продакшене рекомендуется:

1. Использовать **Redis** для хранения кодов 2FA вместо Map в памяти
2. Добавить **rate limiting** для защиты от брутфорса
3. Включить **логирование** всех действий администратора
4. Настроить **мониторинг** попыток входа
5. Использовать **HTTPS** для всех запросов
6. Регулярно **ротировать** токен Telegram бота
