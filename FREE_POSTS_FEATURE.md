# Функция управления бесплатными публикациями

## Описание

Реализована система управления бесплатными публикациями для организаторов:

1. **Админ-панель**: Администратор может устанавливать количество бесплатных публикаций для всех организаторов на главной странице админ-панели
2. **Дашборд организатора**: Организатор видит количество оставшихся бесплатных публикаций на своем дашборде
3. **Логика подсчета**: Бесплатные публикации уменьшаются только при одобрении проекта администратором (когда проект публикуется), а не при отправке на модерацию

## Установка

### 1. Применить настройку в базе данных

Выполните скрипт для добавления настройки по умолчанию:

```bash
npx tsx scripts/apply-free-posts-setting.ts
```

Или выполните SQL напрямую:

```sql
INSERT INTO settings (id, key, value)
VALUES (gen_random_uuid(), 'default_free_posts', '3')
ON CONFLICT (key) DO NOTHING;
```

### 2. Перезапустить приложение

```bash
npm run dev
```

## Как это работает

### Для администратора

1. Зайдите в админ-панель: `/admin/dashboard`
2. В верхней части страницы увидите блок "Бесплатные публикации для организаторов"
3. Нажмите кнопку "Изменить"
4. Введите новое количество бесплатных публикаций
5. Нажмите "Сохранить"

**Важно**: Это значение применяется только к новым организаторам при их одобрении администратором.

### Для организатора

1. После одобрения администратором организатор получает установленное количество бесплатных публикаций
2. На дашборде организатора (`/organizer/dashboard`) отображается карточка с количеством оставшихся бесплатных публикаций
3. При отправке проекта на модерацию счетчик НЕ уменьшается
4. Счетчик уменьшается только когда администратор одобряет проект и он публикуется
5. Если проект отклоняется, счетчик НЕ уменьшается

## Технические детали

### Измененные файлы

1. **API**:
   - `app/api/admin/settings/free-posts/route.ts` - новый API для управления настройкой
   - `app/api/admin/projects/[id]/approve/route.ts` - обновлен для уменьшения счетчика при одобрении
   - `app/api/admin/organizers/[id]/approve/route.ts` - обновлен для установки начального значения

2. **UI**:
   - `app/admin/dashboard/page.tsx` - добавлен блок управления бесплатными публикациями
   - `app/organizer/dashboard/page.tsx` - добавлена карточка с оставшимися публикациями

3. **База данных**:
   - Используется существующее поле `freePostsRemaining` в таблице `organizer_profiles`
   - Добавлена запись в таблицу `settings` с ключом `default_free_posts`

### Логика работы

1. **При одобрении организатора**:
   ```typescript
   // Получаем настройку из базы
   const setting = await prisma.setting.findUnique({
     where: { key: 'default_free_posts' }
   });
   const defaultFreePosts = setting ? parseInt(setting.value) : 3;
   
   // Устанавливаем организатору
   await prisma.organizerProfile.update({
     data: { freePostsRemaining: defaultFreePosts }
   });
   ```

2. **При одобрении проекта**:
   ```typescript
   // Уменьшаем счетчик в транзакции
   await prisma.$transaction([
     prisma.project.update({ data: { status: 'published' } }),
     prisma.organizerProfile.update({
       data: { freePostsRemaining: Math.max(0, current - 1) }
     })
   ]);
   ```

3. **При отклонении проекта**:
   - Счетчик НЕ изменяется
   - Проект возвращается в статус `rejected`
   - Организатор может исправить и отправить заново

## Проверка работы

1. Создайте нового организатора
2. Одобрите его как администратор
3. Проверьте, что у него появилось установленное количество бесплатных публикаций
4. Создайте проект и отправьте на модерацию - счетчик не должен измениться
5. Одобрите проект как администратор - счетчик должен уменьшиться на 1
6. Отклоните другой проект - счетчик не должен измениться
