# Руководство по редиректам авторизации

## Реализованная логика

### Для авторизованных пользователей

Если пользователь уже вошел в систему (имеет активную сессию), он **НЕ МОЖЕТ** заходить на следующие страницы:

- `/` - Главная страница (лендинг)
- `/login` - Страница входа
- `/register` - Выбор типа регистрации
- `/register/volunteer` - Регистрация волонтера
- `/register/organizer` - Регистрация организатора
- `/forgot-password` - Восстановление пароля
- `/forgot-password/verify` - Ввод кода восстановления
- `/forgot-password/reset` - Сброс пароля

**Поведение:** При попытке зайти на эти страницы пользователь автоматически перенаправляется на `/dashboard`

### Для неавторизованных пользователей

Если пользователь НЕ вошел в систему, он **НЕ МОЖЕТ** заходить на следующие страницы:

- `/dashboard` - Личный кабинет
- Любые другие защищенные страницы

**Поведение:** При попытке зайти на эти страницы пользователь автоматически перенаправляется на `/login`

## Реализация

### 1. Middleware (серверная проверка)

**Файл:** `middleware.ts`

Middleware проверяет JWT токен в cookies и выполняет редиректы на уровне сервера:

```typescript
// Пути только для гостей
const guestOnlyPaths = ['/', '/login', '/register', '/forgot-password'];

// Защищенные пути
const protectedPaths = ['/dashboard'];
```

**Преимущества:**
- ✅ Работает до загрузки страницы
- ✅ Защита на уровне сервера
- ✅ Быстрые редиректы без загрузки контента

### 2. Клиентская проверка (дополнительная защита)

**Файлы:**
- `app/page.tsx` (лендинг)
- `app/login/page.tsx`
- `app/register/page.tsx`
- `app/register/volunteer/page.tsx`
- `app/register/organizer/page.tsx`

Каждая страница проверяет авторизацию при загрузке:

```typescript
useEffect(() => {
  const checkAuth = async () => {
    try {
      const response = await fetch('/api/auth/me');
      if (response.ok) {
        router.push('/dashboard');
        return;
      }
    } catch (err) {
      // Не авторизован
    }
    setChecking(false);
  };
  checkAuth();
}, [router]);
```

**Преимущества:**
- ✅ Дополнительная защита на клиенте
- ✅ Показывает индикатор загрузки
- ✅ Плавный UX

## Индикатор загрузки

Пока проверяется авторизация, показывается спиннер:

```tsx
if (checking) {
  return (
    <div className="min-h-screen flex items-center justify-center bg-gray-50">
      <div className="text-center">
        <div className="animate-spin rounded-full h-12 w-12 border-b-2 border-[#00CC00] mx-auto"></div>
        <p className="mt-4 text-gray-600">Загрузка...</p>
      </div>
    </div>
  );
}
```

## Тестирование

### Сценарий 1: Неавторизованный пользователь

1. Откройте браузер в режиме инкогнито
2. Перейдите на `/` - ✅ Должна открыться главная страница
3. Перейдите на `/login` - ✅ Должна открыться страница входа
4. Перейдите на `/dashboard` - ✅ Должен быть редирект на `/login`

### Сценарий 2: Авторизованный пользователь

1. Войдите в систему
2. Перейдите на `/dashboard` - ✅ Должна открыться страница дашборда
3. Перейдите на `/` - ✅ Должен быть редирект на `/dashboard`
4. Перейдите на `/login` - ✅ Должен быть редирект на `/dashboard`
5. Перейдите на `/register` - ✅ Должен быть редирект на `/dashboard`

### Сценарий 3: После регистрации

1. Зарегистрируйтесь как волонтер
2. Подтвердите email
3. Вы должны быть на `/dashboard`
4. Попробуйте вернуться на `/` - ✅ Должен быть редирект на `/dashboard`
5. Попробуйте открыть `/login` - ✅ Должен быть редирект на `/dashboard`

### Сценарий 4: После выхода

1. Нажмите "Выйти" в дашборде
2. Вы должны быть на `/login`
3. Попробуйте открыть `/dashboard` - ✅ Должен быть редирект на `/login`
4. Перейдите на `/` - ✅ Должна открыться главная страница

## Безопасность

### Двойная защита

1. **Middleware** - проверяет на сервере, не дает загрузить страницу
2. **Client-side** - дополнительная проверка, улучшает UX

### JWT токен

- Хранится в httpOnly cookies
- Проверяется на каждом запросе
- Автоматически удаляется при выходе

### API endpoint `/api/auth/me`

Используется для проверки авторизации:
- Возвращает 200 + данные пользователя, если авторизован
- Возвращает 401, если не авторизован

## Возможные проблемы и решения

### Проблема: Бесконечный редирект

**Причина:** Middleware и клиентская проверка конфликтуют

**Решение:** Убедитесь, что пути в `guestOnlyPaths` и `protectedPaths` не пересекаются

### Проблема: Мигание контента

**Причина:** Страница загружается до проверки авторизации

**Решение:** Используется состояние `checking` и показывается спиннер

### Проблема: Редирект не работает

**Причина:** JWT токен не валидный или отсутствует

**Решение:** 
1. Проверьте cookies в DevTools
2. Проверьте JWT_SECRET в .env
3. Попробуйте выйти и войти снова

## Расширение

### Добавление новых защищенных страниц

В `middleware.ts`:

```typescript
const protectedPaths = ['/dashboard', '/profile', '/settings'];
```

### Добавление новых страниц только для гостей

В `middleware.ts`:

```typescript
const guestOnlyPaths = ['/', '/login', '/register', '/forgot-password', '/about'];
```

Затем добавьте клиентскую проверку в компонент страницы.
