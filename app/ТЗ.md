## 1. Общие сведения

### 1.1. Наименование системы
Веб-платформа для организации волонтёрских и социальных проектов (далее — «Платформа»)

### 1.2. Назначение системы
Платформа предназначена для:
- управления волонтёрскими проектами
- учёта волонтёров
- планирования задач
- коммуникации участников
- геймификации (достижения и награды)
- формирования отчётности
- монетизации публикаций проектов

---

## 2. Цели разработки

### 2.1. Бизнес-цели
- Повышение эффективности работы организаторов
- Увеличение количества проектов
- Рост числа волонтёров
- Прозрачность деятельности

### 2.2. Технические цели
- Единое информационное пространство
- Автоматизация процессов
- Предоставление инструментов аналитики и отчётности

## 3. Функциональные требования

### 3.1. Волонтёр

- Регистрация с подтверждением email
- Вход и восстановление пароля
- Заполнение профиля: ФИО, контакты, навыки, интересы, доступное время, город
- Просмотр каталога проектов с фильтрацией
- Подача заявки на проект
- Отслеживание статуса заявки
- Отметка выполнения задачи, получение обратной связи
- История участия
- Календарь задач с контролем пересечений
- Волонтёрская книжка (PDF)
- Чат поддержки с ИИ-ассистентом
- Просмотр достижений и активных наград

### 3.2. Организатор

- Вход по роли
- Создание, редактирование, удаление проектов и задач
- Отправка проекта на модерацию
- Просмотр базы волонтёров, поиск по навыкам
- Подтверждение/отклонение заявок
- Назначение волонтёров на задачи
- Отслеживание прогресса задач
- Коммуникация с волонтёрами (рассылки, уведомления)
- Формирование отчётов (PDF, Excel)
- Чат поддержки с ИИ-ассистентом
- 3 бесплатные публикации при регистрации
- Отображение остатка бесплатных публикаций
- Оплата публикации через Finik (QR или ссылка)
- История платежей

### 3.3. Администратор

- Управление пользователями (роли, блокировка, верификация)
- Модерация проектов (одобрение/отклонение с указанием причины)
- Управление справочниками (навыки, категории)
- Настройка достижений и наград
- Просмотр дашборда с метриками
- Формирование отчётов по платформе
- Журнал аудита действий пользователей
- Массовые рассылки уведомлений
- Просмотр истории чатов поддержки
- Управление партнёрами
- Настройка стоимости публикации и лимита бесплатных публикаций
- Просмотр отчёта по доходам

### 3.4. Геймификация (достижения и награды)

- Автоматическая выдача достижений при выполнении условий:
  - участие в первом проекте («Новичок»)
  - участие в 5 проектах («Активист»)
  - участие в 10 проектах одной категории («Эксперт»)
  - выполнение всех задач за месяц без срывов («Надёжный»)
  - создание собственного проекта («Лидер»)
- За каждым достижением закреплена награда от партнёра (скидка/услуга)
- При получении достижения волонтёр видит награду и срок действия
- Волонтёр показывает экран с достижением партнёру для получения скидки
- Партнёр не выполняет проверок — скидка даётся на доверии
- По истечении срока награда скрывается из списка активных

### 3.5. Уведомления

- Уведомления внутри платформы + email
- Волонтёр: статус заявки, новая задача, дедлайн, новое достижение
- Организатор: новая заявка, отметка выполнения, отказ, дедлайн, успешная оплата
- Комментирование задач
- Массовые рассылки от администратора

### 3.6. Аналитика и отчётность

- Волонтёр: волонтёрская книжка (PDF)
- Организатор: отчёт по проекту (участники, часы, статусы, Burndown Chart) в Excel/PDF, история платежей
- Администратор: сводный отчёт по платформе, воронка проектов, дашборд с метриками, отчёт по доходам

### 3.7. ИИ-поддержка

- Чат-бот на базе LLM через API
- Обработка запросов на русском языке
- Классификация обращений
- Персонализация ответов (роль, имя)
- Доступ к базе знаний (FAQ, руководства)
- Автоматическое переключение на администратора при низкой уверенности
- Аналитическая панель для администратора

### 3.8. Платежи (Finik)

- Оплата публикации проекта при исчерпании бесплатных публикаций
- Интеграция через API/SDK
- Динамический QR-код или платёжная ссылка
- Webhook со статусом `SUCCEEDED`
- Верификация webhook секретным ключом
- Хранение данных платежа в БД

## 4. Нефункциональные требования

### 4.1. Архитектура

| Компонент | Технологии |
|-----------|------------|
| **Backend** | Java 17+, Spring Boot 3.x, Express.js (TypeScript) |
| **Frontend** | TypeScript, React, HTML5/CSS3, Bootstrap |
| **БД** | PostgreSQL |
| **API** | REST |

### 4.2. Безопасность

- HTTPS
- JWT-токены + Spring Security
- Хеширование паролей (BCrypt)
- Ролевая модель (волонтёр, организатор, администратор)
- Валидация ввода на клиенте и сервере
- Верификация webhook от Finik

### 4.3. Производительность

| Операция | Время |
|----------|-------|
| Реакция на действия пользователя | 0.5–2 секунды |
| Формирование сложных отчётов | 1–10 секунд |

### 4.4. Совместимость

| Тип | Требования |
|-----|-------------|
| **Сервер** | Windows Server 2016+/Linux, 4-8 ГБ ОЗУ, Java 17+/Node.js 20+, PostgreSQL 19+ |
| **Клиент (ПК)** | Windows 8–11, macOS, Ubuntu; Chrome, Firefox, Edge, Opera |
| **Клиент (мобильные)** | Android 6+, iOS 9+; Safari, Chrome, Firefox |

### 4.5. Интерфейс

- Адаптивный дизайн
- Единый стиль (спокойные тона: синий, зелёный, белый)
- Интуитивно понятный UI
- Графики и диаграммы для аналитики

### 4.6. Языки интерфейса

- Русский (с возможностью добавления кыргызского и английского)
